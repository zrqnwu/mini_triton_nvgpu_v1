# 两天读通项目主线最小清单

更新时间：2026-04-06

适用项目：

- `/home/zhangruiqi/mini_triton_nvgpu_v1`

适用对象：

- 只有基础 `MLIR` 和基础 `CUDA`
- 希望先用最短时间把项目主线读通

---

## 一、目标

这份清单的目标不是：

- 2 天学完整个领域
- 2 天看完所有 `.cpp`

而是：

- 2 天内先把这个项目的主线读顺
- 知道每一层大概负责什么
- 知道为什么 pass 顺序不能乱
- 知道哪些文件以后值得继续深挖

读完后，你至少应该能口头讲清楚：

1. 从 `tb.matmul` 到 PTX，大致经过哪些层
2. 为什么 `tb.pipeline_mainline` 很关键
3. 为什么不能一上来先看 `LowerPipelineToNVGPU.cpp`

---

## 二、总原则

这 2 天只做下面 3 件事：

1. 先看总主线
2. 再看边界
3. 最后再看少量关键合同

这 2 天不要做的事：

- 不要逐行啃大 `.cpp`
- 不要一开始追所有实现细节
- 不要一上来先纠结 `split-k / persistent`
- 不要先啃全部 Triton 源码

---

## 三、第 1 天：先把总结构读顺

## 上午：看项目总目标和总 pipeline

先看这 3 个文件：

1. [README.md](/home/zhangruiqi/mini_triton_nvgpu_v1/README.md)
2. [Stage1 数据结构与主线 Pass 讲解](/home/zhangruiqi/mini_triton_nvgpu_v1/docs/stage1_data_structures_and_passes.md)

这一步只回答 3 个问题：

1. 当前项目只做什么范围
2. 全部 pass 顺序是什么
3. 为什么它不是一个 lowering 文件全包

读完后，你要自己写出这一条链：

```text
tb.matmul
  -> KernelConfig / TargetInfo
  -> MatmulSemantics
  -> ProgramMappingPlan
  -> EncodingPlan
  -> TransportPlan
  -> AccumulatorPlan + EpiloguePlan
  -> MatmulRewritePlan
  -> BufferModel
  -> LoopPlan
  -> LatencyPlan
  -> PipelinePlan
  -> AsyncPlan
  -> PipelineExpansion
  -> tb.pipeline_mainline
  -> LowerPipelineToNVGPU
  -> LowerEpilogueVectorIO
```
